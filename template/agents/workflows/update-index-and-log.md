# Workflow: Update Index and Log

## Purpose

Keep the content catalog and semantic timeline current.

## Update index when

- A page is created.
- A page becomes an important entry point.
- A major page description changes.

## Append log when

- Ingest changes the wiki.
- Query output is filed back.
- Lint fixes structural issues.
- A major concept decision is accepted.

## Done when

- New and important pages are findable from `index.md`.
- Substantial changes are recorded in `log.md`.
