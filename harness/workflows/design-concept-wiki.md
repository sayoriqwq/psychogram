# Workflow: Design Concept Wiki

## Purpose

为一个输入概念设计独立 concept wiki 的边界和初始结构。

## Read first

1. [[../glossary]]
2. [[../lead]]
3. [[../agents/roles/template-architect]]
4. [[../index]]

## Steps

1. 明确 concept 的名称、边界、作者用途和非目标。
2. 判断它是否应该生成独立 wiki；若包含多个概念，先拆分。
3. 选择 template 中的通用 page type，不添加领域专用结构，除非该 instance 明确需要。
4. 草拟 `wiki/lead.md`、guide 入口、初始 glossary 和核心问题。
5. 标出任何需要作者确认的边界。

## Done when

- 概念边界清楚。
- 初始 wiki 结构可独立存在。
- 没有把 fixture 或 harness 内容混入 concept 内容。
